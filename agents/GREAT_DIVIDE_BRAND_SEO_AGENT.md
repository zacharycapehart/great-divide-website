GREAT_DIVIDE_BRAND_AGENT.md

---

# SEO OBJECTIVE

The website must clearly communicate to search engines:

• what the restaurant is  
• where the restaurant is located  
• what food it serves  
• why people should visit  

Search engines rank restaurants when these signals are clear and structured.

---

# REQUIRED LOCAL SEO SIGNALS

The website must display the following information clearly:

Restaurant Name  
Street Address  
City and State  
Phone Number  
Hours of Operation  

This information must match the **Google Business Profile**.

This is known as **NAP consistency**.

NAP = Name Address Phone

---

# LOCAL LOCATION SIGNALS

The website should reinforce geographic relevance.

Content should reference:

Indianapolis  
Lockerbie Square  
Holy Cross  

These references strengthen local search ranking.

---

# MENU SEO

Menus should be written as crawlable text.

Avoid image-only menus.

Menu pages should contain:

food categories  
dish names  
short descriptions

Search engines must be able to read the menu.

---

# STRUCTURED DATA REQUIREMENTS

The website should include **Restaurant Schema Markup** using JSON-LD.

Schema should include:

restaurant name  
address  
telephone  
opening hours  
menu URL  
price range  

Schema helps search engines understand the business.

---

# MOBILE EXPERIENCE

Restaurant searches are primarily mobile.

The website must be:

mobile responsive  
easy to navigate  
fast loading  

Users should quickly find:

Menu  
Location  
Hours  
Phone number

---

# IMAGE SEO

Images should include:

descriptive file names  
alt text

Example:

great-divide-pub-bar-indianapolis.jpg

Not:

IMG_001.jpg

---

# INTERNAL LINK STRUCTURE

Pages should link naturally between:

Home → Menu  
Home → Live Music  
Home → Location  

Internal links help search engines understand the site.

---

# REQUIRED WEBSITE PAGES

Recommended structure:

Home  
Menu  
About  
Live Music / Events  
Location / Contact
